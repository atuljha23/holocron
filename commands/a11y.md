# /holocron:a11y

Target: `$ARGUMENTS` — or the working-tree diff if no arg.

## Gather

- If no arg: `git diff --name-only` filtered to UI files (`.tsx`, `.jsx`, `.vue`, `.svelte`, `.html`, `.astro`).
- If arg is a file: that file.
- If arg is a directory: UI files under it (at most 20 — if more, ask the user to narrow).

## Delegate

Spawn `@a11y-auditor` on the selected files. Feed it the files and ask for:

- Blocker / Serious / Minor findings
- WCAG success criterion (or clear reason) per finding
- Concrete fix per finding, with file:line

## Output

Group findings by severity. If a component is clean, say so — do not invent findings.

## After

If the user wants durable rules from this pass (e.g. "we always set `lang` on `<html>`"), suggest `/holocron:learn`.
